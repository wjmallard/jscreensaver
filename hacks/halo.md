# halo — port notes

Port of `halo.c` by Jamie Zawinski (1993) — circular interference patterns. A few **halos** (concentric ring families) sit on the screen, their centres slowly drifting, and every step all of them breathe outward by one ring-spacing. Where two families overlap the rings **cancel**, and that cancellation is what carves the moire fringes out of the overlap.

Original: <https://www.jwz.org/xscreensaver/> · source: `xscreensaver-6.15/hacks/halo.c` (458 lines).

## Algorithm
Each circle has a centre `(x,y)`, a `radius`, a ring spacing `increment`, and a drift `(dx,dy)`. Increments are rolled with a triangular bias toward small values (so spacings are mostly tight). Each step: every circle's current **filled disk** is XOR-accumulated into a persistent buffer, then its radius grows by `increment`. Summed over the growing radii the XOR carves concentric **rings** (each `increment` wide); the families expand together, and where two families overlap the rings cancel.

A breath ends (`done`) — but only ever on an **odd** step (`iterations & 1`) — when a family either collapses to points (`radius < 0`, after a reversal) or grows large enough to fully enclose the screen rectangle (an analytic corner-containment test). On `done` the state machine picks one of:
- **Re-pick** (when we've just breathed all the way in, `increment < 0`): the screen is blank, so re-seed the centres and shift the palette cursor.
- **Inside-restart** (sometimes, `clearTick == 0 && rand(3)==0`): wrap every radius back near zero (`radius %= increment`) and arm `clearTick` to an odd 4..11 — the figure regrows from the inside, and a full wipe lands a few breaths later.
- **Reverse** (otherwise): negate each `increment` and back the radius up, so the families breathe back inward.

`clearTick`, once armed, counts down on subsequent `done`s and wipes the buffer to black when it reaches zero. With `animate`, instead of reversing, the centres random-walk (bouncing off the edges) and radii wrap, so the moire slides around.

## Shared skeleton
Same as every port — see [[squiral]] for the full description: a no-build ES module exporting `title` + `start(canvas) → { stop, pause, resume, reinit, config, params }`, an rAF **lag-accumulator** loop in place of the C's `usleep(delay)` (identical pace at any refresh rate, with a catch-up cap), `devicePixelRatio` folded into sizes/line-widths, and the config box driven by the inline `params` schema via `config-box.js`. Closest technique twins: [[spiral]] (sparse moire rings) and [[braid]] (stroked concentric arcs).

## Deviations from the C
- **XOR in a pixel-exact 1-bit plane (the headline technique).** Canvas has no XOR raster op, so the C's `GXxor` (`merge_gc`) is reproduced over a **typed-array depth-1 plane** — `frameBits`/`bufBits`, each pixel exactly `0` or `255` — where `^=` IS XOR (`255^255=0`, `255^0=255`), so overlaps cancel exactly and the moire is exact. Colour is **never** mixed into this plane (a coloured cancel would not cancel and would break the moire); it is applied only at blit time (below). Crucially the disks are **scan-converted by hand** with X11's center-inside-radius rule (a pixel is set iff `(x−cx)²+(y−cy)² ≤ r²`, integer spans, **zero antialiasing**) — so consecutive frames' edges line up to the pixel and cancel completely. (An intermediate version used `ctx.arc().fill()` + a threshold + `globalCompositeOperation='difference'`; that *thinned* but never removed a ~1px residual rim, because canvas fill rounds edges ~1px differently as the radius grows. The typed-array rasteriser removes it entirely.)
- **Filled disks XOR-accumulated, not stroked outlines.** Each step the union of this step's **filled disks** (`XFillArc`) is scan-filled into `frameBits`, then `bufBits ^= frameBits`; summed over the growing radii that yields concentric **`increment`-wide filled annuli**, and the XOR *between* families is the interference. (An earlier version stroked 1px ring *outlines* and claimed it was "mathematically equivalent / the same picture" — it is **not**: outlines are ~2× too dense and draw edges only, never the filled bands.) Only the C's `DEBUG` quadrant copies are dropped.
- **Faithful palette via `colormap.js`.** The C builds its colormap **once** in `halo_init`: `make_uniform_colormap` (a one-way hue ramp 0→359 at a random 66–100% S,V) most of the time, else `make_smooth_colormap` (the muted 2–5-anchor HSV loop) — `random()%(seuss?2:10)` non-zero picks uniform, so seuss is ½ uniform, ramp 9⁄10. This port uses `makeColorRampRGB` / `makeSmoothColormapRGB` with those exact odds and that once-per-init cadence (an earlier version used a single fixed vivid HSL rainbow); only the fg/bg cursors move during a run. **seuss** colourises the `bufBits` plane exactly like `copy_gc` (set bits → `colors[fgIndex]`, clear bits → `colors[bgIndex]`, advancing on each inner-collapse re-pick); **ramp** over-paints filled disks straight onto the canvas in `GXcopy` (opaque), only while breathing *in*, cycling the colour each step. `random` resolves to **ramp** 1/4 of the time else **seuss**; `ncolors ≤ 2` is mono (white/black, forces seuss); `ramp` disables `animate`.
- **Two typed-array planes, not the full X plumbing.** The C's `pixmap` + `buffer` are the `frameBits` + `bufBits` `Uint8Array`s (above); the `XCopyPlane` colourised blit becomes a per-pixel recolour of `bufBits` into the visible `ImageData` (`bufBits[i] ? fgPacked : bgPacked`, the same packed-`Uint32` write used across the 2D ports). The `DEBUG` quadrant copies and manual expose repair are dropped.
- **`delay` default = the stock 100000 µs** (the slider maps 1:1 to `hacks/config/halo.xml`). **Framerate calibration: no `OVERHEAD` applied.** At 100000 µs the port's nominal 10 fps already matches the live binary's measured **8.7 fps** within the `-fps` overlay's ~15% noise — a slow hack, so the framework overhead is a negligible fraction (see the framerate-calibration note). The earlier by-eye 60000 µs default is dropped. Mode-dependent timing is faithful: **seuss + animate** runs at `delay/100` (~100× faster), and **ramp** inhibits the sleep on the (undrawn) out-breath. Range/labels otherwise map 1:1 to `hacks/config/halo.xml` (`count` 0–20, `colors`/ncolors 1–255, `mode`, `animate`).
- **Retina scaling matches the C.** Only the ring `increment` is multiplied by `st->scale` (3× on >2560-px canvases); the centre drift `dx/dy` is **not** scaled, and the auto halo-count is taken in **device** px — both as in the C.
- **Encoding:** the µs unit in the Frame-rate slider is written as the escape `µs` (ASCII-safe rule); the only non-ASCII bytes in `halo.js` are em dashes in comments.

## Correctness self-review
The brief warns that parse-clean + right-contract isn't enough — subtle reset/closure bugs have shipped. Traced by hand:
- **Termination/closure.** `done` is gated on an **odd** `iterations`, exactly as the C, so a breath can't end on the wrong parity. The enclose-the-screen test ports the C's four-corner `(x/r)² + (y/r)² < 1` check verbatim; `radius === 0` is skipped (eschew the divide), `radius < 0` ends the inward breath. Every `done` branch re-seeds what the next step reads: **re-pick** rebuilds `circles` and the palette cursor, **inside-restart** wraps every radius and arms an *odd* `clearTick`, **reverse** negates `increment` and backs the radius up. There is no float-equality closure test, so nothing can silently fail to fire.
- **No freeze / no runaway.** The radius always moves (`+= increment`, increment ≠ 0 by construction since `(inc+3)*scale ≥ 3`), so a family always reaches a `done` and the cycle turns over; the loop can't wedge. `wrapMod` guards a zero divisor so a degenerate increment can't `NaN` a radius. The lag-accumulator caps catch-up at `MAX_CATCHUP_STEPS = 8`, so a backgrounded tab can't burst.
- **First frame looks right.** `init()` seeds radii from `nrand(increment)` (small, non-degenerate) and clears to black, so the very first breath already draws on-screen rings — no off-screen or zero-radius start.
- **pause / resume / reinit.** `pause()` cancels rAF and parks `rafId = 0`; `resume()` resets `lastTime = 0` before re-arming, so no catch-up jump. `reinit()` re-runs `init()` (re-seeds + clears to black) for a clean fresh screen after a non-live config change.

**Spot-check in the browser:** confirm the moire reads as **filled, `increment`-wide bands** whose overlaps cancel into dark interference nodes — not 1px ring outlines, and not a single colour: **seuss** should show two palette colours (fg/bg) that shift on each re-pick, **ramp** concentric colour rings. Check that centres stay on-screen and that after a long breathe-out the screen wipes/regrows rather than freezing.
