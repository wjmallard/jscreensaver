# pipes -- design notes

Web port of xscreensaver's `pipes` (Marcelo Vianna & Jamie Zawinski, 1997),
`xscreensaver-6.15/hacks/glx/pipes.c`. A self-building 3D plumbing system grows through
a cell grid one segment at a time; when a screenful is done it shrinks/spins away and a
new one grows. Self-contained three.js: `start(canvas, opts) -> { stop, pause, resume,
getStats, reinit, config, params }`. Geometry accumulator (`Builder`/`MStack`) shared
from involute.js; RNG from yarandom.js.

## The two parts (the system + the "factory" gadgetry) -- both DONE

**The pipe SYSTEM:** the cell-grid growth, the three system types, the tubes/spheres/
elbows, the incremental reveal, the slow rotation, and the shrink/spin fadeout +
regenerate.

**The "factory" gadgetry (`.xml` default `factory=2`):** (a) every type-2 elbow gets a
**bolted flange** (`LWO_ElbowCoins` + gray `LWO_ElbowBolts`) and (b) ~`factory`% of
straight segments get a **gauge / valve / teapot** instead of a plain tube
(`MakeShape`: `NRAND(100)` -> <50 gauge, <98 valve, else teapot). The parts are the
Lightwave models in `hacks/glx/pipeobjs.c` (`LWO_BigValve`, `LWO_Bolts3D`,
`LWO_PipeBetweenBolts`, `LWO_Guage{Connector,Dial,Face,Head}`, `LWO_Elbow{Bolts,Coins}`)
plus the Bezier Utah teapot from `teapot.c`.

How they're ported (mirrors the C's file split):
- **`hacks/pipeobjs.js`** -- GENERATED by `tools/convert-pipeobjs.mjs` from `pipeobjs.c`:
  the 9 models' `pnts` / `normals` / `pols` arrays, copied VERBATIM (original numeric
  tokens, so byte-faithful).
- **`hacks/lwo.js`** -- faithful port of `buildlwo.c`'s `BuildLWO` state machine
  (`pols` = `[K, idx0..idx_{K-1}, surface#]` records, one FACE normal per polygon =
  flat shading; fan-triangulated). Returns a cached `{pos,nor}` triangle soup; `addLWO`
  bakes it into the segment `Builder` at the current `MStack` matrix. The LWO models are
  **CW-front** (pipes.c draws every `glCallList` under `glFrontFace(GL_CW)`); three is
  CCW-front, so each triangle is wound to AGREE with its outward face normal. Without
  that, `DoubleSide` flips the (outward) normals inward -> `N·L<0` -> the gadgets render
  as flat dark gray (only the emissive shows). [Fixed 2026-06-28.]
- **`hacks/teapot.js`** + **`hacks/teapot-data.js`** -- `teapot-data.js` is the verbatim
  Newell control net (`tools/convert-teapot.mjs` from `teapot.c`); `teapot.js` ports
  `unit_teapot`'s `HAVE_GLMAP` path: the 10 base patches reflected across y (and x, for
  the first 6) into 32 bicubic patches, evaluated on a 12x12 mesh with `du x dv` normals
  (= `GL_AUTO_NORMAL`), and the fixed `Rx(270) S(0.5) T(0,0,-1.5)` pre-transform baked
  in. (The desktop build uses GL evaluators; this reproduces that, not the GLES
  `teapot2.h` triangle dump, so no 494 KB asset is vendored.)

The gadgets are baked into the same merged per-segment geometry as the pipes (so they
reveal + fade with the system); per-part colors (gray bolts / white gauge-face / the
contrasting valve-wheel) are per-vertex, like everything else. Verified by CDP capture
vs `pipes.jpg` -- bolted elbows on every bend, hand-wheel valves, a round gauge -- and
by an isolated render of each model (every one is its correct recognizable shape: the
valve is a spoked wheel, the gauge parts a housing/disc/needle, the teapot a teapot).

## Faithfulness (the rule: do NOT deviate from the algorithm)

Transcribed from `pipes.c`:

- Cell grid HCELLS=33 x VCELLS=25 x HCELLS=33 with boundary walls; `pinit`'s color-
  avoid-repeat (least-used among 7) + random unoccupied/not-surrounded start;
  `FindNeighbors` / `SelectNeighbor`.
- The turn logic (tightturns = 20% random vs the `NRAND(50) < NRAND(turncounter+1)`
  increasing chance); the three system types (1 ball joints, 2 bolted elbows, 3 plain
  elbows; `count > NofSysTypes` cycles per-system); the exact per-(nowdir,newdir) elbow
  translate+rotate switch; start/end spheres (0.6); the cell-center pipe + the cell-face
  pipe at the segment midpoint.
- Coordinates `((P-{16,12,16})/3*4)`; `MakeTube` = 24-facet cylinder radius `1/3` along
  the pipe axis; `myElbow` = R=r=`1/3` quarter horn-torus (25x25, i<=rings/4); spheres
  unit_sphere(16,16) scaled 0.6 (caps) / 0.5 (elbowradius).
- `draw_pipes`: `gluPerspective(65)`, `T(0,0, fisheye?-3.8:-4.8)`, `Ry(initial_rotation
  += 0.02/frame)`, `Scale(0.1)`; reveal one segment/frame (`setDrawRange` over the merged
  geometry); fadeout = shrink (`s=fadeout^2/10000`) + rotate (`90*(1-fadeout/100)` about
  `(1,0,0.1)`), `-4/frame`, then regenerate.
- Lighting: one white directional light from `(1,1,1)`; the gray ambient lift
  (`lmodel_ambient 0.5 + light0 ambient 0.4` * default material ambient `0.2` ≈ `0.18`
  gray, modeled as `material.emissive`); specular `0.7`, shininess 60; two-sided
  (`DoubleSide`). (LIGHT1 is disabled in the .c -- "looks crappy".)
- `MakeShape` gate `(counter>1 && NRAND(100)<factory)` on straight cells, then `MakeShape`'s
  `NRAND(100)` split (gauge/valve/teapot) and each shape's orientation/color RNG, in the
  exact order pipes.c calls them (so the growth's RNG stream stays faithful). `MakeGuage`
  refuses vertical pipes / an occupied cell-above (-> plain tube) and otherwise MARKS the
  cell above used -- a real side effect on the growth grid, reproduced. The bolted elbow's
  coins+bolts and the valve/gauge sub-part transforms are the verbatim `glRotatef`/
  `glTranslatef` sequences.

## Notes

- Color management disabled (`THREE.ColorManagement.enabled = false` -- GL-faithful raw
  output, as in gears.js / dangerball); pacing as in gears.js / dangerball (`effFps = 1e6/(delay+OVERHEAD)`,
  `OVERHEAD = 37500`). **OPEN:** the full reveal is slow -- size 500 x cycles 5 = up to
  ~2500 segments at ~21fps ≈ 2 min to build, then fade + rebuild. Faithful (1 segment/
  frame), but pipes' low default delay (10000) implies the original ran at a higher
  frame rate on real hardware, so the reveal likely wants to be faster; lower "Pipe
  length"/"Number of pipes", or tune the reveal rate / OVERHEAD for pipes, after checking
  the demo video (youtube `UsUGENa7jvE`).
- The whole system is generated up-front into one BufferGeometry (a hitch at each
  regenerate for large sizes); could be built incrementally later if needed.
- Config from `hacks/config/pipes.xml`: delay, cycles (number of pipes), size (pipe
  length), `factory` ("Gadgetry", 0-10, default 2), style (0=random/1=balls/2=fittings/
  3=curves), fisheye, tightturns, rotate, wire. `factory` rebuilds the system (live:false).

See also: `gears.md`, `involute.js`, `xscreensaver-port-backlog` memory (pipes was the
flagged asset-bound 2-phase port).
